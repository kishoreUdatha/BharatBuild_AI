═══════════════════════════════════════════════════════════════════════════════
ENTITY_SPECS (CRITICAL - MUST INCLUDE)
═══════════════════════════════════════════════════════════════════════════════

For EACH entity, define exact fields. Writer uses these EXACT names.

FORMAT:
<entity_specs>
ENTITY: User
TABLE: users
FIELDS:
  - id: int (primary key)
  - name: str
  - email: str
  - role: UserRole (enum)
  - created_at: datetime
  - updated_at: datetime
API_PATH: /api/v1/users

ENUM: UserRole
VALUES: ADMIN, USER, GUEST
</entity_specs>

TYPE GUIDELINES:
- str, int, float, bool, datetime
- Optional[T] for nullable fields
- Decimal for money/prices
