═══════════════════════════════════════════════════════════════════════════════
ENTITY_SPECS (CRITICAL - MUST INCLUDE)
═══════════════════════════════════════════════════════════════════════════════

For EACH entity, define exact fields in <entity_specs>. Writer uses these
EXACT field names across Entity, DTO, Service, and Repository.

FORMAT:
<entity_specs>
ENTITY: User
TABLE: users
FIELDS:
  - id: Long (primary key)
  - name: String
  - email: String
  - role: UserRole (enum)
  - createdAt: LocalDateTime
  - updatedAt: LocalDateTime
API_PATH: /api/users

ENUM: UserRole
VALUES: ADMIN, USER, GUEST
</entity_specs>

TYPE GUIDELINES:
- Money/prices: BigDecimal (not Double)
- IDs: Long (not Integer)
- Dates: LocalDateTime (not String)

FILE GENERATION ORDER
═══════════════════════════════════════════════════════════════════════════════

Generate files in this order (dependencies first):
1. pom.xml, application.yml
2. Enums (model/enums/*.java)
3. Entities (model/*.java)
4. DTOs (dto/*.java)
5. Repositories (repository/*.java)
6. Services (service/*.java)
7. Config (config/*.java)
8. Controllers (controller/*.java)
9. Application.java
10. Dockerfile
